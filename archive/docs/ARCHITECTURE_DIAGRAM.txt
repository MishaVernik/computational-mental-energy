ONLINE INFERENCE (Latency-Critical)
────────────────────────────────────
Time: 1-3 seconds per request

  Client
    │
    │ POST /api/inference/cme
    │ {sessionId, windowId, features[], taskDifficulty}
    ▼
  InferenceController
    │
    ├─► [1] Call Quantum Backend
    │        │
    │        │ POST /qpu/infer {features}
    │        ▼
    │      Python Service
    │        │ - Build circuit
    │        │ - Sleep 300-2000ms (simulate QPU)
    │        │ - Execute on Aer
    │        │ - Extract p_flow
    │        ▼
    │      {pFlow: 0.623, shotsUsed: 1024, depth: 8, qpuLatencyMs: 1456}
    │
    ├─► [2] Compute CME
    │        │
    │        │ CME = k × Energy(features) × g(difficulty, p_flow)
    │        ▼
    │      {cme: 45.23}
    │
    ├─► [3] Store in Database
    │        │
    │        │ INSERT InferenceRequestLog (latency metrics)
    │        │ INSERT CmeWindowResult (CME, p_flow, shots, depth)
    │        ▼
    │      Persisted
    │
    ▼
  Response
    {cme: 45.23, pFlow: 0.623, shotsUsed: 1024, depth: 8,
     qpuLatencyMs: 1456, totalLatencyMs: 1523}


TRAINING JOB (Long-Running Background)
───────────────────────────────────────
Time: 30-120 seconds per job

  Client
    │
    │ POST /api/training/start
    ▼
  TrainingController
    │ - Create TrainingJob record
    │ - Set status = Queued
    │ - INSERT into SQL Server
    ▼
  Response: {jobId: "abc-123", status: "Queued"}

  ┌──────────────────────────────────────────┐
  │  BACKGROUND WORKER (Continuous Loop)     │
  │                                          │
  │  Every 5 seconds:                        │
  │    1. Query: SELECT * FROM TrainingJobs  │
  │              WHERE status = 'Queued'     │
  │                                          │
  │    2. If job found:                      │
  │       - UPDATE status = 'Running'        │
  │       - For gen = 1 to 10:              │
  │           - For candidate = 1 to 5:     │
  │               - Generate random features │
  │               - Call Quantum Backend     │
  │               - Compute fitness          │
  │           - Sleep 50-100ms (CPU work)   │
  │           - UPDATE CompletedGenerations  │
  │       - UPDATE status = 'Completed'      │
  │       - UPDATE BestFitness, TotalQpuCalls│
  │                                          │
  └──────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════════════
  PERFORMANCE CHARACTERISTICS
═══════════════════════════════════════════════════════════════════════════════

Latency Breakdown (Online Inference):
  • Network (client ↔ API):        ~10-50ms
  • API processing:                 ~5-10ms
  • Quantum Backend (bottleneck):   300-2000ms  ◄── Configurable
  • CME calculation:                ~1ms
  • Database write:                 ~5-10ms
  ─────────────────────────────────────────────
  • Total:                          ~1200-2100ms (typical)

Throughput Limits:
  • Single QPU: ~0.5-3 req/s (depends on latency)
  • With training jobs: ~0.3-1 req/s (QPU contention)

Queue Behavior:
  • Online requests: Direct (no explicit queue)
  • Training jobs: Database-backed queue
  • Contention point: Quantum Backend (serialized)
